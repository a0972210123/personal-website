## 1. 課文

### 1.1 商業模式=價值的三部曲

一句話定義：**商業模式回答「價值如何被創造（create）、交付（deliver）、捕獲（capture）」**。產品只是中間一環——歷史上最賺錢的創新往往不在產品，在模式：吉列把「賣刮鬍刀」改成「送刀架賣刀片」、Xerox 把「賣影印機」改成「按張計費租賃」、Rolls-Royce 把「賣飛機引擎」改成「按飛行小時收費（Power-by-the-Hour）」——產品沒變，捕獲價值的方式重寫了整個產業。

### 1.2 九宮格（Business Model Canvas）：一頁講完一門生意

Osterwalder 的九宮格是全球通用語，九格分左右兩半：

**右半＝價值面（前台，對應 M2 行銷月）**
| 格 | 問句 | 接哪一週 |
|---|---|---|
| 客群 CS | 為誰創造價值？ | W6 STP |
| 價值主張 VP | 幫他完成什麼任務？ | W5 JTBD |
| 通路 CH | 怎麼觸及與交付？ | W8 漏斗 |
| 顧客關係 CR | 一次性？訂閱？社群？ | W7 付費心理 |
| 收入流 R$ | 錢從哪進來、怎麼收？ | 本週 1.4 |

**左半＝效率面（後台，對應 Q3 營運季）**
| 格 | 問句 | 接哪一週 |
|---|---|---|
| 關鍵資源 KR | 靠什麼資產？（含資料飛輪——W25） | W16 VRIO |
| 關鍵活動 KA | 必須擅長做什麼？ | W27–28 |
| 關鍵夥伴 KP | 什麼外包給誰？ | W32 build vs buy |
| 成本結構 C$ | 錢往哪流？固定/變動比例？ | W11 |

九宮格的力量在**一頁可比**：把 Spotify 與唱片行畫在同一張紙上，模式差異一目了然；也在**一頁可辯**——九格之間必須說得出因果（「訂閱制收入流」要求「持續交付價值的關鍵活動」與「留存導向的顧客關係」——格子互相鎖定，一格改動全盤連動）。

### 1.3 HBR 四要素：診斷「何時需要新模式」

Johnson–Christensen–Kagermann 的精煉版四要素，重點在**互鎖**：

1. **顧客價值主張（CVP）**：幫目標顧客把一件重要的事做得更好、更便宜、更方便
2. **利潤公式**：收入模式 × 成本結構 × 單位毛利要求 × 資源週轉速度——W11 的語言在此升格為模式設計參數
3. **關鍵資源**與 4. **關鍵流程**：交付 CVP 的人、技術、流程

**需要「新模式」（而非新產品）的五個訊號**：①有機會服務被既有模式排除的大群體（低價破壞——W18 的模式版）；②有機會把新技術包進新模式（或把既有技術帶進新市場）；③有機會把「任務」帶到還沒人聚焦任務的地方（W5）；④需要抵禦低端破壞者；⑤競爭基礎正在遷移（W18 的性能過剩後）。

經典案例：**Hilti**（工具機廠）從「賣電鑽」轉成「工具車隊管理訂閱」——工地要的任務是「工具隨時可用」不是「擁有工具」（W5 的洞），CVP 重寫→收入流從一次性變月費→關鍵資源從通路變車隊管理系統與直銷團隊——四要素連鎖重寫，這才叫商業模式創新（只改定價不改資源與流程，多半陣亡）。

### 1.4 收入模式選型錄（每種都有成立條件）

| 模式 | 成立條件 | 帳怎麼算（W11 工具） |
|---|---|---|
| 一次買斷 | 交易型任務、低回購頻率 | 看單筆毛利 |
| 訂閱 | 持續性任務、價值隨時間交付 | LTV/CAC、流失率是生死線 |
| 抽成/平台 | 先解「雞蛋問題」（兩邊都要有人） | 抽成率 × GMV；網路效應到位前燒錢 |
| Freemium | 邊際服務成本≈0＋自然轉付費路徑 | 轉付費率（常見 2–5%）× 免費者服務成本 |
| 廣告 | 巨大流量＋注意力可賣 | eCPM；「用戶不是顧客，是商品」的倫理與體驗代價 |
| 刮鬍刀模式 | 耗材綁定＋鎖定機制（專利/生態） | 主體可虧，耗材毛利要蓋回來 |

**選型的第一性原理**：收入模式要**順著任務的自然形狀**——任務是持續的（每週學管理）→ 訂閱順；任務是一次性的（搬家）→ 硬做訂閱是折磨顧客。其次才是財務偏好（訂閱平滑現金流、買斷回收快）。

### 1.5 完成檢驗：兩道關卡

1. **因果通順**：九格連起來講一個 60 秒的故事，每格都是上一格的必然——講不順的地方就是設計漏洞
2. **數字兜攏**：利潤公式落地成 W11 的單位經濟表——CVP 再美，LTV/CAC < 1 就是慈善事業。九宮格是敘事，單位經濟是算術，**兩者互相審查**（W10「敘事與數字」的紀律又來了——它會在 W36 估值時third次出現，這不是巧合，是商業分析的通用骨架）

## 4. Quiz

**Q1（單選）** Rolls-Royce 把噴射引擎改成「按飛行小時收費」，主要重寫了九宮格的：
- A. 客群與通路
- B. 收入流、顧客關係與關鍵活動（從賣斷變長期服務與監控維護）
- C. 只有價格
- D. 關鍵夥伴

**Q2（單選）** Freemium 模式成立的兩個關鍵條件是：
- A. 產品免費且行銷預算大
- B. 服務免費用戶的邊際成本趨近零＋存在自然的免費→付費升級路徑
- C. 競爭者也採用 freemium
- D. 有創投資金支持燒錢

**Q3（單選）** 「先解雞蛋問題」指的是哪種模式的先天難關？
- A. 訂閱制
- B. 刮鬍刀模式
- C. 平台抽成模式——買方沒來賣方不來，賣方沒來買方不來
- D. 一次買斷

**Q4（簡答）** 「收入模式要順著任務的自然形狀」——用兩個例子（一正一反）說明，並解釋逆著任務形狀收費會在哪些指標上現形（提示：W11、W7）。

**Q5（簡答）** Hilti 從賣工具轉訂閱制，為什麼「只改定價、不改資源與流程」的版本注定失敗？用四要素互鎖說明至少兩條斷裂點。

<details>
<summary>👉 展開解答與解說</summary>

**Q1：B。** 收入流（賣斷→按時計費）帶動顧客關係（交易→長期績效夥伴，航空公司把引擎妥善率外包給你）與關鍵活動（製造→遠端監控、預測性維護、備品調度）。注意這正是「格子互鎖」：改一格而不改配套格＝模式沒改成，只是換了發票格式。

**Q2：B。** 兩個條件缺一不可：邊際成本不趨近零，免費用戶會吃垮你（實體產品做 freemium=自殺）；沒有自然升級路徑（免費版太好用或太難用），轉付費率撐不起全體成本。算帳公式：轉付費率 × 付費 LTV ＞ 全體用戶服務成本 ＋ CAC。

**Q3：C。** 平台的冷啟動困境。標準解法：先單邊補貼（Uber 補司機）、自己充當供給（Airbnb 創辦人自己拍房源照片——W19 的禮賓式 MVP 兼冷啟動）、或從利基雙邊小圈子起步（W6 灘頭堡的平台版）。

**Q4 參考答案：** 正例：Netflix 訂閱制順「持續娛樂」任務——每月都在交付價值，月費自然。反例：婚紗攝影做訂閱制=逆任務形狀（一生一兩次的任務），顧客感到被綁架。逆形狀的現形指標：流失率異常高、LTV 算不出穩態（W11）、顧客抱怨「沒用到還被扣款」（付費痛感錯位，W7 Gourville–Soman：付費節奏與消費節奏脫鉤過頭）、NPS 崩。反過來，把持續性任務做成一次買斷也虧：把終身價值一次賤賣（W38 的 VC 會問你為什麼不做 recurring revenue）。

**Q5 參考答案：** 斷裂一（利潤公式 × 關鍵資源）：訂閱制的現金流是細水長流，但 Hilti 要先墊資買下整批工具車隊（資產負債表劇變）——沒有配套的融資與資產管理能力，現金流當場斷裂（W9 成長吃現金的模式版）。斷裂二（CVP × 關鍵流程）：承諾「工具隨時可用」需要全新流程——車隊追蹤系統、預防性維修、直銷團隊談長約——原本「賣斷給經銷商」的流程與人才完全不會做這些；只改報價單，第一個大客戶的工具趴窩那天，承諾就破產。一般式：**商業模式是四要素的連鎖重寫，「只改收費方式」的模式創新是把訂閱貼紙貼在舊引擎上。**
