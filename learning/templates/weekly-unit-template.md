# 週次單元標準格式

月檔內每個週次單元依此格式撰寫；模型產出教學（含單元學習網站）時，內容結構也對應這些欄位。

```markdown
### Week N：單元標題

- **本週問題**：一句話的驅動問題（教學開場用，引發好奇）。
- **學習目標**：2–3 條，用「能夠……」開頭，可檢核。
- **核心概念**：400–600 字中文講解。這是講義骨架，模型展開成課文；專有名詞附英文。
- **必讀原典**：1–2 個，格式 `作者，〈篇名〉，出處（年份）— 連結`。
  頂尖商學院或原創者一手來源；paywall 標注「（需訂閱／有免費額度）」。
- **延伸資源**：0–2 個中文資源（譯本書、影片、文章），選讀，落後時先砍這欄。
- **實作練習**：1 個，30–60 分鐘，能連結 Matt 自身情境（個人網站、投資專欄、創業題目）最好。
- **自我檢核**：2–3 個問題，模型出 quiz 用；答案應能從核心概念＋必讀推得。
- **標籤**：`3R:天時|地利|人和|基礎` ＋ `職能:行銷|人資|生產|財務|研發|總務|資訊|管理`（可複選）。
```

## 單元學習網站（模型產出時）的最低規格

1. 單頁、自包含（無外部 CDN），行動裝置可讀。
2. 區塊順序：本週問題 → 學習目標 → 課文（核心概念展開）→ 必讀原典卡片（外連）→ 實作任務 → 互動 quiz。
3. 所有引用附來源連結（引流原則）；quiz 提供即時對錯回饋與解說。
4. 底部標注週次、3R 維度、職能標籤，與 `00-overview.md` 一致。
