# curriculum/ — 一年期投資學習地圖（給 AI 模型的使用說明）

> **這個資料夾不是給人直接閱讀的。** 它是寫給未來 AI 模型（Claude 或其他）的結構化課綱資料，
> 目的是讓模型能逐週指導 owner 學習，並產出「每週單元」形式的學習內容或學習網站。

## 目的

把 owner（葉淨維 Matt）在一年內培養成具備三種角色基礎能力的人：

1. **優良投資人**——能獨立分析公司、估值、配置資產
2. **基金操盤手**——能管理投資組合、衡量風險與績效、遵守決策紀律
3. **組織財務長（CFO）**——能做資本預算、融資與股利決策、管理公司財務

同時透過每週的「教家人」環節，把知識轉譯成家人聽得懂的語言——這既是家人教育，
也是 owner 的理解測驗（教得出來才算真懂），並與本站 `family-investing` 專欄系列連動。

### units/ 的產出慣例

- 命名：`week-NN-slug.md`（NN 兩位數週次，slug 為英文主題短名，例：`week-01-tvm.md`）
- 單元一經產出即為 owner 的實際教材；已產出的單元要修改時，用 PR 讓 owner review
- 每個單元固定五區塊：**講義／必讀導讀／練習／教家人腳本／完成檢核**（練習計算題附文末解答）
- 單元開頭的引言必須回答「本週在三個角色中的定位」
- 每個單元講義開頭安排 10 分鐘「回補上週教家人卡住的點」（第 1 週除外）

## 如何產出一個「每週單元」

1. 讀 `01-overview.md` 的進度表，找到目前要產出的週次 N
2. 讀對應季度檔中「第 N 週」的完整 section
3. 依 section 內容產出單元（網頁、文件或對話式教學皆可），必須包含：
   - **講義**：把「核心知識點」展開成 owner 可在 ~2 小時內讀完的教材，深度以「學習目標」為準
   - **必讀導讀**：告訴 owner 必讀資料的重點與閱讀順序（連結從 `resources.md` 以資源 ID 展開）
   - **練習**：把「練習」欄位具體化成可交付的任務，附解答或檢核方式
   - **教家人腳本**：一段 5–10 分鐘、家人聽得懂的講稿或活動設計
4. 總份量對應 **每週 3–5 小時**（必讀 ~2h＋練習 ~1h＋教家人 ~30m；選讀為額外）
5. 單元完成後，在 `01-overview.md` 該週的「完成」欄填入 `✅ YYYY-MM-DD`，並在「心得」欄記一行 owner 的回饋

## 規則與紀律

- **語言**：一律繁體中文（zh-TW），專有名詞附英文原文
- **連結**：只引用 `resources.md` 裡登記過的來源；需要新來源時，先加進 `resources.md`（給 ID）再引用
- **連結驗證**：本課綱撰寫時的環境無法連外驗證連結（皆為長期穩定的 canonical URL）。
  首次實際使用某連結時請先驗證；若失效，優先找**同作者／同機構的官方替代來源**，並回寫更新 `resources.md`
- **與專欄連動**：週次標註「專欄 #N」者，先讀 `src/content/blog/family-investing-N-*.md` 避免重複；
  「教家人」環節的產出可轉化為新的專欄文章（開新 PR，遵守根目錄 `CLAUDE.md` 的工作流程）
- **順序**：週次有前置依賴（各週標註「前置」欄位），不要跳週；owner 要求加速時，可將相鄰兩週合併為一個大單元
- **調整**：owner 的興趣或時間改變時，可以改排程——但要同步更新 `01-overview.md`，讓下一個接手的模型看到最新狀態
- **評量**：每季最後一週（13、26、39、52）是實作 capstone，不是新知識；驗收標準寫在該週的「完成檢核」

## 內容產出的品質要求

- 所有公式、數據、歷史事件必須正確；不確定時查證或標註不確定性
- 用台灣視角補充在地細節（台股交易制度、稅制、常用工具），但理論以國際教材為準
- 每個單元開頭要一句話回答「這週學的東西，在三個目標角色中的哪裡會用到」
- 教家人腳本的語言難度對齊 `family-investing` 專欄既有文章（無金融背景的成年家人）
